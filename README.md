# Why I Built This

I originally built this framework after repeatedly encountering the same analytical failure in local service businesses, agencies, and small marketing teams: everyone optimized marketing campaigns using numbers that were technically correct but operationally false.

A campaign would appear successful inside Google Ads:

| Metric      | Platform Report |
| ----------- | --------------- |
| Spend       | $15,843         |
| Conversions | 480             |
| CPA         | $33             |

The business owner would conclude:

> "Our marketing is working."

However, after reconciling those same campaigns against CRM records and completed jobs, the actual business outcome looked very different:

| Metric          | CRM-Audited Reality |
| --------------- | ------------------- |
| Qualified Leads | 95                  |
| Completed Jobs  | 24                  |
| Real CPA        | $166                |
| Real ROAS       | 3.05×               |

The problem wasn't inaccurate software.

The problem was an analytical failure: marketing platforms optimize for recorded events, while businesses survive on completed transactions and collected revenue.

This project productizes a repeatable reasoning framework:

1. Start with claimed marketing performance.
2. Trace every conversion into operational systems.
3. Remove disqualified, duplicated, and non-commercial events.
4. Recalculate acquisition economics using business reality.
5. Prioritize interventions by financial impact.

Instead of building another dashboard, I built a reusable audit methodology that answers a simpler question:

> **"Which marketing dollars actually produced revenue?"**

### Three Traps That Catch Even Experienced Marketing Teams

#### Trap #1 — Platform Conversion Inflation

**Decision made:**

Increase budget because CPA appears low.

| Metric      | Platform | Audited |
| ----------- | -------- | ------- |
| Spend       | $15,843  | $15,843 |
| Conversions | 480      | 95      |
| CPA         | $33      | $166    |

**Why this reasoning fails:**

Advertising platforms optimize for recorded events rather than business outcomes.

**Correct approach:**

Calculate acquisition economics only after CRM reconciliation.

**Correct decision outcome:**

Reduce budget allocation and redesign conversion tracking.

<details>
<summary>Formula logic</summary>

```excel
=IFERROR((Reported_Conversions-Real_Leads)
/Reported_Conversions,0)
```

```excel
=IFERROR(Total_Spend/Matched_Real_Leads,0)
```

</details>

---

#### Trap #2 — Optimizing High-Volume Low-Quality Keywords

**Decision made:**

Increase bids on high-volume search terms.

| Search Term          | Spend | Leads | Decision |
| -------------------- | ----- | ----- | -------- |
| emergency plumber    | $220  | 12    | Keep     |
| free plumbing advice | $310  | 0     | Exclude  |

**Why this reasoning fails:**

Click volume measures attention, not commercial intent.

**Correct approach:**

Evaluate search terms using qualified lead production.

**Correct decision outcome:**

Exclude non-commercial search patterns.

<details>
<summary>Formula logic</summary>

```excel
=IF(
 AND(
   Spend>(Max_CPA*1.5),
   Leads=0
 ),
 "CRITICAL WASTE",
 "KEEP"
)
```

</details>

#### Trap #3 — Budget Allocation Based on Reported ROAS

**Decision made:**

Transfer budget toward apparently successful campaigns.

| Campaign   | Reported ROAS | Real ROAS |
| ---------- | ------------- | --------- |
| Campaign A | 7.8×          | 1.4×      |
| Campaign B | 3.2×          | 4.5×      |

**Why this reasoning fails:**

Revenue attribution breaks when operational confirmation is absent.

**Correct approach:**

Calculate ROAS from financial records.

**Correct decision outcome:**

Reallocate spend toward campaigns with verified revenue production.

<details>
<summary>Formula logic</summary>

```excel
=IFERROR(
 Real_Revenue
 /
 Total_Spend,
0)
```

</details>

### Example Scenario

A residential HVAC company spends $18,400 monthly across eight Google Ads campaigns.

Initial reporting shows:

| Metric               | Value   |
| -------------------- | ------- |
| Ad Spend             | $18,400 |
| Reported Conversions | 562     |
| Reported CPA         | $32.74  |

After importing CRM data:

| Metric          | Value   |
| --------------- | ------- |
| Qualified Leads | 118     |
| Booked Jobs     | 31      |
| Revenue         | $57,200 |
| Real CPA        | $155.93 |
| Real ROAS       | 3.11×   |

Further analysis reveals:

* 67% of reported conversions never entered CRM.
* Three campaigns generated 72% of actual revenue.
* Twenty-two search terms consumed $3,480 with zero booked jobs.
* One "free HVAC training" keyword group consumed $940 alone.

Recommended actions:

1. Pause two underperforming campaigns.
2. Add eighteen negative keywords.
3. Reallocate 35% of budget toward verified campaigns.
4. Replace front-end conversion tracking with CRM validation.

Expected annual savings:

```
Recovered Waste:
$3,480/month × 12
=
$41,760/year
```
